dos alcances, dos modos de falla
El tracing ve lo que corrió. Los contratos de exit code ven lo que no corrió.

Tracing y observabilidad

alcance: lo que se ejecutó
Spans, tokens, latencia, reintentos. Muy útil para los caminos que el agente recorrió.
  • tokens por tarea
  • latencia de tool-call
  • tasa de retry / error
  • un proceso que nunca arrancó
  • un cron que no se disparó
  • un agente que salió con exit 0 sin trabajar

Contrato de exit code + heartbeat

alcance: lo que debió correr
Cada agente declara: "voy a pingar éxito cada N horas." Ping faltante = aviso.
  • ¿arrancó siquiera?
  • ¿terminó con éxito?
  • exit 0 / 1 / 2 mapeados a causas
  • heartbeat de 24h → dead man's switch
  • el wrapper de cron registra el exit code
  • healthchecks.io o systemd OnFailure=
la observabilidad agrega visibilidad · los contratos agregan responsabilidad · necesitas ambos
