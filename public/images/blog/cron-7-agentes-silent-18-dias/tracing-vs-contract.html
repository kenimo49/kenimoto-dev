dois escopos, dois modos de falha
Tracing vê o que rodou. Contratos de exit code veem o que não rodou.

Tracing & observability

escopo: o que foi executado
Spans, tokens, latência, retries. Excelente para os caminhos por onde o agente passou de fato.
  • tokens gastos por tarefa
  • latência de tool-call
  • taxa de retry / erro
  • processo que nunca subiu
  • cron que não disparou
  • agente que saiu com exit 0 sem fazer nada

Contrato de exit code + heartbeat

escopo: o que deveria ter rodado
Cada agente declara: "vou pingar sucesso a cada N horas." Ping faltando = página.
  • chegou a rodar?
  • terminou com sucesso?
  • exit 0 / 1 / 2 mapeados a causas
  • heartbeat 24h → dead man's switch
  • wrapper de cron loga exit code
  • healthchecks.io ou systemd OnFailure=
